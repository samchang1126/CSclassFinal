🩺 智慧健康日誌與風險評估系統 (Smart Health Log & Risk Assessment System)
📌 專案簡介 (Project Overview)
本專案是一套專為個人健康自我管理量身打造的全端 AI 智慧儀表板系統 。系統整合了 Java Spring Boot 後端、內嵌式高訊號 H2 關聯式資料庫、Tailwind CSS 響應式前端，以及基於 C4.5 演算法思想的機器學習決策樹（Decision Tree）商務邏輯分類模型 。

使用者能動態記錄每日的關鍵健康指標（睡眠時數、當日步數、心情分數） ，系統會即時透過多層分支決策樹精準演算出其健康風險等級（低／中／高） ，並提供極具商務價值的個人化多用戶趨勢追蹤儀表板 。

🌲 核心技術亮點 (Technical Highlights)
1. 真正具備層次感的 C4.5 多層決策樹推理
拒絕單層 If-Else 規則：系統實作真正的多層特徵切分（先判斷特徵 A，再依結果判斷特徵 B），完美呈現決策樹演算法的層次感。


高訊號種子資料驗證：預灌 10 位使用者共 900 筆高度去隨機化、具備強烈規律的歷史種子數據。經 C4.5 資訊增益（Information Gain）計算，精準找出黃金切分閾值（睡眠: 6.0hr、步數: 5000步、心情: 4分） 。


中間邊界情境處理：模型能完美處理複合型邊界狀況，例如「步數與活動量完全正常（如 8000 步），但因情緒低落（如 2 分）導致的中間風險（中風險）」，商務分類邏輯精準不翻車 。

2. 數據視覺化進化：帶有「風險識別」的三軸折線圖

Chart.js 點位色彩連動：歷史趨勢折線圖上的每一個健康數據圓點（Data Point），會根據決策樹當時判定出來的風險等級，動態著色為 🟢 綠色（低風險）、🟡 黃色（中風險）或 🔴 高風險 。

觸控懸停反饋 (Tooltip)：當游標移至折線圖點位時，系統會自動透過 Callback 函數喚醒提示框，動態回顯當日的決策樹分類結果。

3. 商務多用戶動態篩選架構
表單內建 user_id 商務欄位，歷史紀錄列表與 Chart.js 圖表支援「顯示全部人」與「個別使用者 ID」的非同步即時切換與動態重新渲染，滿足多人健康管理的商業應用場景 。

📊 系統架構與資料流向 (Architecture & Data Flow)
本專案採用前後端不分離、包在同一個 Java 專案部署的極速高穩定架構 ，整體資料流向如下：

[前端 index.html 表單] ──(JSON Payload)──► [POST /health-logs]
       │                                         │ (呼叫決策樹模型)
       ▼                                         ▼
[自動非同步無縫刷新] ◄──(撈取最新紀錄)─── [GET /health-logs] ◄──► [H2 資料庫儲存]
       │
       └─► [Chart.js 渲染] ──► 依風險等級動態著色圓點 (紅/黃/綠)
       └─► [決策面板點亮]  ──► 視覺化展示「睡眠 ➔ 步數 ➔ 心情」多層分支軌跡

網頁初始化 (GET)：載入時自動呼叫 GET /health-logs，自 H2 資料庫撈出全用戶歷史紀錄，動態抽取不重複 ID 注入篩選器，並繪製初始折線圖 。


表單提交 (POST)：送出今日指標後，前端發送非同步 POST /health-logs 請求 。後端模型計算完風險等級後寫入關聯式資料庫 ，並將結果即時回傳 。

動態軌跡高亮：前端 JavaScript 接收到分類結果後，會將決策樹第一層（睡眠分支）、第二層（步數分支）到第三層（心情分支）的通過狀態在畫面上高亮點亮，直觀展示機器學習模型的推理脈絡。


極致防呆機制：若遇後端斷線或異常，前端 try-catch 會引導網頁自動收起舊數據、回復乾淨狀態並跳出錯誤提示，絕不造成畫面凍結 。

🛠️ 開發環境與依賴庫 (Tech Stack)

Backend: Java 17+, Spring Boot 3.x, Spring Data JPA 


Database: H2 In-Memory Database (記憶體資料庫) 


Frontend: HTML5, Tailwind CSS (CDN), 原生 JavaScript (Vanilla JS), Chart.js 


Build Tool: Maven (pom.xml) 


IDE & AI Tools: Cursor (Vibe Coding / Prompt Engine), GitHub 

🚀 部署說明 (Deployment)

CI/CD 自動化連動：專案已成功託管至 GitHub 遠端儲存庫，並與 Zeabur 雲端平台 進行自動化自動部署連動 。


一鍵上線：每次本地端執行 git push 後，雲端平台將會自動偵測 pom.xml 並自動跑完 Maven 打包編譯，實現零設定、免跨域（CORS）問題的一鍵免密碼全端上線！
